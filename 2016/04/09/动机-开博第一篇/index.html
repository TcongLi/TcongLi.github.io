动机--开博第一篇

发表于 2016-04-09   |   分类于 随想   |  

引子

你说你懂得二元一次，我便做了这方程与你看；
你说题目繁琐也终有答案，我便解了这公式与你看；
你说暖暖旧日好时光，我便留下回忆与你看……
数学女孩 —结城浩

If you can quit, then you quit; If you can not quit,you are a 【writer】.

在折腾了各种记录媒介之后,发现还是这种MARKDOWN(不用写成吨的html尖括号)+GIT(可以版本控制)+hexo(提供简洁漂亮的模版,还可以一件部署到github,方便易用)+gitpages(免费) 比较适合用来写一些东西.
在这之前,用过:

  1. 笔+纸
    • 优点: 显而易见,可以随身携带,在灵感出现的瞬间将其捕获,无需专注格式,可以用最自然的方式随手画出各种思维导图,流程图.
    • 局限: 同样显而易见
      1. 难以在段落中间增,删,以及修改内容
      2. 难以传播,在纸上写出来的的东西难以以美观清晰的形式展现在网页,pdf,以及打印出来
      3. 难以保存,很多在纸上即兴写出来的东西难以很好的保存下来,有的写在纸上,有的写在本子上,难以系统的归档
      4. 缺少系统性,层次性以及完整性,有时候需要丰富一个自己的知识体系,这就需要各种形式的表现形式(图片,代码块,等等),而这些往往是一页A4纸难以承载的,而多页更是难以分出层次,逻辑上十分不清晰
      5. 无法备份
  2. gist,word,国际排版工具Tex系列,云笔记系列
    • 优点:
      1. gist:带有版本控制功能,可以设置为public和secret,方便添加,修改,分享方便
      2. word:通用,基本上人人都会用,而且适合打印
      3. Tex系列:开源,具有令任何文字排版工具羡慕的精准排版,数学公式也是非常美观
      4. 云笔记系列:可以实现移动端与服务器同步
    • 局限:
      1. gist:更像是一页一页的便签,没有层次,而且需要科学上网才能访问
      2. word:臃肿,排版复杂,写作时需要考虑各种与写作无关的东西,难以分享,与云笔记之间兼容性差
      3. Tex系列:学习成本高,对中文支持差
      4. 云笔记系列:臃肿,复杂,并且难以实现 网页<–>纸<–>客户端 之间的格式转换

下面介绍目前这种方式的优点:

  1. 创作者直接接触到的是通用性极强的【文本编辑器】+【markdown语法】,可用自己喜欢的文本编辑器(不管是跨平台的emacs,vim还是windows下我最喜欢的notepad++).可移植性,可扩展性极强,首先,文本格式的内容可以与几乎【任何】平台,【任何】应用兼容,在文本编辑器下不用考虑字体,颜色,底纹以及word上各种奇怪但不实用的排版,这些排版可以用简单的markdown语法来实现.创作者直接接触的是直观性仅次于纸和笔的工具,精力集中在创作的内容本身.而编辑好的内容可以用pandoc,以及各种能够解释md语法的工具转换成网页,word,PDF等等,而PDF可以实现与word,ppt,html的互相转换.
  2. 可以用git来进行版本控制,像控制代码一样控制创作出来的东西,从而避免了自己每次写完word后另存为一个新版本最后发现你的硬盘上全是xxxV1.0.0.doc,xxxV2.0.0.doc(而且版本之间关联性差,无法实现回退,分支).
  3. 用github免费托管静态主页,自由度极高,hexo可one-command deploy(一键部署)到github
  4. 细心的读者可能会发现,使用这种方式的四个模块完全是高内聚,低耦合的,即相互的依赖关系很小,从最简单的文本,到最终的博客主页,是一个按精致程度递进的关系,你完全可以只写纯文本格式的记录,也可以选择从纯文本格式到用markdown标记,再到发布到博客,或者直接转成PDF打印.后来的模块都是加在前一个模块基础上的,这就像自己盖房子,可以选择只盖一层,也可以选择盖五层;可以选择用木头搭建,也可以选择用钢筋混凝土,至于程度,完全取决于自己的时间和精力,而云笔记之类就像需要你买下一整套精致的别墅,虽然精致,但对其依赖性太强,难以实现自己的私人订制.

WHY?

WHY[0]—为什么不用微博?

如今互联网,微博异常火爆,而博客异常萧条,微博不光写起来快,读起来也快——迎合了很多人喜欢“快餐文化”的心态,然而
这些所谓”快餐文化”【真的】好吗?

1.微博的本质

  1. 简短性:传统的博客犹如写文章,写之前通常需要构思,需要做些准备工作;而写微博就像说一句话一样容易.因此,”短小”这个特点导致了写微博的门槛很低——这就让大量的网民能参与其中.
  2. 双向性:跟博客相比,微博更多的是”多对多”的模式,在社交平台上,任何人可以互粉,(对于大V来说,该模式可能变为”一对多”),而这种模式也导致了信息的高速流动.
  3. 公开性:相比于朋友圈,微博可以实现信息更强的 公开性 ,这就导致了某条微博(特别是某个大V发布的)能在最迅速的时间内火爆到段子满天飞(譬如优衣库事件,可以在晚上几个小时内火到人人皆知(虽然在第二天早上起来早已不见干货XD)),但是这种相比朋友圈更强的公开性,也导致了一条微博如果在短时间内没爆红的话,随即就会被迅速淹没在大海般的网络信息流里:)

2.微博的缺点

  1. 信息过于破碎
  2. 信息过于肤浅,缺乏深度
  3. 信息真实性不能保证
  4. 读者容易淡忘
  5. 时效性太短
  6. 生命周期太短
  7. 信噪比太差

3.摆正读/写微博的心态

“不要担心错过某条消息——会被你错过的消息,多半不重要,不看也罢;真正重要的消息,你自然不会错过.”
—编程随想

在微博网站上,有不少“话痨”型的用户.这种人可以几年如一日,每天发几十条甚至上百条消息.而且发的消息,大都是些鸡毛蒜皮的个人琐事.
如果你不幸属于这类人,你得好好反思反思了——你花这么多时间,写了这么多微博,到底有多少人看?即便有人看,能给他们带来什么帮助吗?

WHY[1]—为什么传统博客没有过时?

何谓过时?

过时: 出现了一个完备的替代品

何谓完备?

完备: 新东西具备老东西的【所有】用途和优点,同时可以避免老东西的【某些】缺点.

1.微博无法替代博客: 前文已述.

2.书籍无法替代博客:

在各类媒体中,书籍是最值得和博客相提并论的,在深度和系统化方面,书籍甚至可以比博客做的更好.但书籍也有它的局限性:

  1. 写: 创作的门槛比较高(并不是所有人都有时间,精力,能力写出一本在既系统化又有深度的书).
  2. 读: 时间上,现代人的生活趋向于快节奏的”快餐文化”,往往是在短时间内读一些碎片化的东西,而在一个周末的午后,找个安静的书桌,静下心来,沏上一杯清茶,读一本几百页的好书似乎已然成为一种极其奢侈的享受了;毅力上,读一本技术类的书籍,和读一本小说可大有不同,读一本小说完全可以几十秒钟过一页,而读一本技术类的书籍,往往需要边实践边阅读,同时记下大量对思路进行梳理的笔记,画出清晰明朗的思维导图,以对自己的知识结构进行一个明确的构建,一页可能要反反复复钻研几天,而针对这一页的内容,只是冰山一角,可能又要牵扯到更多更多的相关知识以及前驱知识.而这些知识往往需要自己搜集权威的文献,对这些人而言,哪怕读一本薄薄的,100页左右的书,都是难以完成的任务.如此一来,书籍在一篇好的结构清晰的博客面前,就显得相形见绌了.
  3. 源: 并不是所有书都是好书,中国的出版界太乱,很多作者,很多出版社,往往把利益放在第一位,在这个充满性自由,成功学,拜金的暴戾社会里,出书作者往往深陷糟粕文化泥潭而不知,写出来的东西根本就是对民众的误导,把民众的价值观引向错误的方向,这些作者在蛊惑民心上做出了不可磨灭的贡献;再谈出版社,很多出版社在校正方面并不费很大精力,读者见到错误的观点,错字,往往以后再难有机会改正;在技术书籍方面(尤指IT),大部分中国作者往往是照着优秀的英文文献上断章取义,随意摘取一些观点,而且还重复造轮子,而有些翻译外文书籍的作者往往文笔不会太好,翻译出来势必变味道.

WHY[2]—为什么写传统博客的人越来越少?

相当多的博主已经【丧失】了写长篇博文的能力

1.注意力的碎片化

互联网应用由于持续极度碎片化的内容,必然导致使用者的注意力也变成碎片化的.
当你的注意力变得碎片化,你就难以长时间专注某个事情—-当你无法长时间集中注意力,你甚至连一篇两页的博文都看不完.

2.无法脱离”心里舒适区”

何为 “心里舒适区”?

心里舒适区(Comfort Zone): 用来描述一种心理状态,处于这种心理状态,你会感觉到放松,没有压力和焦虑;如果你处于某个环境中,在该环境下所需要的行为你都可以轻松搞定.那么,你就处于“心理舒适区”.

让自己停留在心里舒适区,大多是为了打发时间.在这种状态下,你不论是在微信上转发一篇心灵鸡汤,抑或在推特上发一条推文,都是感到很舒适的.
与之相比:
要写出一篇有质量的,有一定篇幅的原创博文,是一个【艰苦】的过程.
似乎有相当多的人,每天沉在”心里舒适区”的焦油坑,再也无法脱离其中.

WHY[3]—动机->为什么要写博客?

1.自我解剖—-形成和了解自我

长期写博客最大的好处之一就是,写着写着,你的自我会越来越清晰,你最终会了解自己是一个什么样的人,以及清楚自己想要的是什么,以及自己的优点和不足,自己的人生观,价值观都会越来越清晰明了.
正如我在边栏简介里引用沈从文沈老墓志铭上的一句话:

照我思索,能理解”我”;照我思索,可认识”人”


正是因为我太不了解自己了,想要从深入思考中解剖自己,看清自己的本质

2.思考—-深入而系统

要想写出有质量的原创博文,深入思考是必要条件(但不是充分条件)
除了思考,还有一个必要条件—知识

对某个领域的知识了解不够,显然写不出一篇有质量的原创博文,在写每个领域的话题之前,都需要搜集资料,搜集资料的过程,也是学习的过程,也是扩大知识面的过程.有助于作者重新思考该领域的知识体系,而且有助于发现在该领域知识盲点.
学习程度有深浅之分,学的浅了,自以为懂了,会了,掌握了,其实是囫囵吞枣,那一项技能的很多关键部分都可能会在潜意识里被大脑偷换概念或者引入错误的假设,而在实践中就会暴露出来.反思是一个【最好】的方法,要经常总结和反思学过的知识,把思考的过程写下来,找出各种隐含在潜意识的因果关系.

3.表达能力—-硬伤

写的博文越多,对自己博文质量要求也就越高,文字表达的提升就越明显,还顺带着提升口头表述能力,因为口头表达跟文字表达是相通,都要求你的头脑具有层次性和条理性.
这点本人深有体会,有时候明明脑子里想的一大堆东西,等到了说的时候却难以用简洁的语言说清楚,甚至他人会对我的观点产生误解.归根结底,都是表达能力欠佳,一直以来写作都不是我的擅长,但是人总是要进步的.

4.对抗惰性

人人都有惰性,本人不仅有惰性,还有拖延症,而且是重度拖延症.从小养成的坏习惯,总是先把容易的事情做完,再做相对困难的事情;先把感兴趣的事情做完,再做不感兴趣的事情.结果往往到最后根本没精力去攻克困难的事情了,总是拖延到最后(如deadline的前一天晚上)才草草做完本应该是最需要重视的事情(甚至做不完).

以前一直相信,“最后的总是最好的”

上面这句话当作美味的鸡汤吧,愿君喝完能从教训中学到一些东西.

总结

好的习惯会给人带来持续的效益,良性循环.
第一篇文章写的可能有点粗糙,但确实是看了不少优秀的文章,融合了不少人的观点,记录下了思考的过程.
以后会多写一些技术性的,或是好玩的,但必须是有价值的文章,不显示标明[转载]的文章,均为原创,不限于随笔,随想,对事件的深刻剖析,等等..
如果你觉得我的观点和你产生了共鸣,或者有什么【建设性意见】的话,【欢迎】在评论区留下你的观点,或者邮箱联系,也欢迎转载,但请【务必】标明出处及本文地址.
